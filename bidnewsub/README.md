# AI-Powered Procurement Bidding Evaluation — BTP Account Setup

## Overview

This sample shows how to setup your SAP BTP account for the Discovery Center Mission - [AI-Powered Procurement Bidding Evaluation](https://discovery-center.cloud.sap/index.html#/missiondetail/4327/) for your Enterprise BTP Account.

## Content of setup

The setup is split into two Terraform steps.

### Step 1 — BTP Infrastructure (`step1/`)

Uses the `SAP/btp` provider only.

- Creation of SAP BTP subaccount (or reuse of existing via `subaccount_id`)
- Custom IDP trust configuration
- Role collection assignments: **Subaccount Administrator**, **Subaccount Service Administrator**
- Creation of Cloud Foundry environment and CF org
- Service entitlements:
  - SAP AI Core (`aicore`)
  - SAP HANA Cloud (`hana-cloud`)
  - SAP HANA schemas & HDI containers (`hana`)
  - Object Store (`objectstore`)
  - XSUAA (`xsuaa`)
  - Destination (`destination`)
  - HTML5 Application Repository (`html5-apps-repo`)
- App entitlements and subscriptions:
  - SAP AI Launchpad (`ai-launchpad`)
  - SAP Build Work Zone, standard edition (`SAPLaunchpad`)
  - SAP HANA Cloud Tools (`hana-cloud-tools`)
- Role collection assignments: **Launchpad_Admin**, **SAP HANA Cloud Administrator**
- Auto-generation of `step2/terraform.tfvars` with CF outputs (API URL, Org ID, etc.)

### Step 2 — Cloud Foundry & Services (`step2/`)

Uses both `SAP/btp` and `SAP/cloudfoundry` providers.

- Import and management of default SAP IDP trust configuration (disables SAP ID Service login when a custom IDP is active)
- Creation of the CF dev space
- CF org role assignments: **organization_user**, **organization_manager**
- CF space role assignments: **space_manager**, **space_developer**
- SAP AI Core managed service instance (`extended` plan)
- SAP AI Core service key (credential binding)

## Deploying the resources

Make sure that you are familiar with SAP BTP and know both the [Get Started with btp-terraform-samples](https://github.com/SAP-samples/btp-terraform-samples/blob/main/GET_STARTED.md) and the [Get Started with the Terraform Provider for BTP](https://developers.sap.com/tutorials/btp-terraform-get-started.html)

To deploy the resources you must:

### Setup Step1

1. Set your credentials as environment variables

   ```bash
   export BTP_USERNAME='<Email address of your BTP user>'
   export BTP_PASSWORD='<Password of your BTP user>'
   ```

2. Go into folder `step1` and change the variables in the `step1.tfvars` file to meet your requirements

   > The minimal set of parameters you should specify (besides user_email and password) is global account (i.e. its subdomain) and the used custom_idp and all user assignments

   > Keep the setting `create_tfvars_file_for_step2 = true` so that a `terraform.tfvars` file is created which contains your needed variables to execute setup `step2` without specifying them again in sample.tfvars there.

3. In folder `step1` you initialize your workspace:

   ```bash
   terraform init
   ```

4. You can check what Terraform plans to apply based on your configuration:

   ```bash
   terraform plan -var-file="step1.tfvars"
   ```

5. Apply your configuration to provision the resources:

   ```bash
   terraform apply -var-file="step1.tfvars"
   ```

6. Verify e.g., in BTP cockpit that a new subaccount with a SAP HANA Cloud and SAP Build Work Zone subscriptions have been created.

### Setup Step2

7. Set your CF credentials as environment variables (sensitive — not stored in tfvars), use the custom IDP credentials, please use the credential of defaul IDP .

   ```bash
   export CF_USER='<Email address of your BTP user>'
   export CF_PASSWORD='<Password of your BTP user>'
   ```

   Also set `cf_origin` in `terraform.tfvars` to match your IDP:
   - Custom IDP: `cf_origin = "techrig-platform"` (first segment of `custom_idp` + `-platform`)
   - Default SAP ID Service: `cf_origin = "sap.ids"`

8. Navigate into the `step2` directory and initialize your workspace:

   ```bash
   terraform init
   ```

9. You can check what Terraform plans to apply based on your configuration:

   ```bash
   terraform plan -var-file="terraform.tfvars"
   ```

   > `terraform.tfvars` is auto-generated by step1 when `create_tfvars_file_for_step2 = true`. You can also use `step2.tfvars` if you have one.

10. Apply your configuration to provision the resources:

    ```bash
    terraform apply -var-file="terraform.tfvars"
    ```

11. Verify in BTP cockpit and CF that the dev space, org/space roles, AI Core service instance, and service key have been created.

With this you have completed the quick account setup as described in the Discovery Center Mission - [AI-Powered Procurement Bidding Evaluation](https://discovery-center.cloud.sap/index.html#/missiondetail/4327/).

## Cleanup

To remove all provisioned resources, destroy step2 first, then step1:

```bash
# In step2/
terraform destroy -var-file="terraform.tfvars"

# In step1/
terraform destroy -var-file="step1.tfvars"
```
